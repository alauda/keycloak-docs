# Configure Multi-Factor Authentication

This guide describes how to enable and configure Multi-Factor Authentication (MFA) for a Keycloak Realm using Time-based One-Time Password (TOTP).

## Prerequisites

- A running Keycloak instance with a configured Realm.
- Access to the Keycloak Admin Console with realm admin privileges.
- Users have an authenticator app installed (for example, Google Authenticator, Microsoft Authenticator, or FreeOTP).

## Step 1: Configure the OTP Policy

Set the OTP policy for the Realm:

1. Log in to the Keycloak Admin Console.
2. Select the target Realm.
3. Go to **Realm Settings** > **OTP Policy**.
4. Configure the following settings:

| Setting | Recommended Value | Description |
|:--------|:------------------|:------------|
| **OTP Type** | `Time Based (TOTP)` | TOTP generates a new code every 30 seconds. |
| **OTP Hash Algorithm** | `SHA1` | The hashing algorithm used for OTP generation. `SHA1` provides the broadest authenticator app compatibility. |
| **Number of Digits** | `6` | Standard 6-digit OTP code. |
| **Look Ahead Window** | `1` | Number of time windows to check before and after the current one to account for clock drift. |
| **OTP Token Period** | `30` | Token validity period in seconds. |

5. Click **Save**.

## Step 2: Require OTP in the Authentication Flow

Configure the browser authentication flow to require OTP after username and password:

1. Go to **Authentication** > **Flows**.
2. Select the **Browser** flow (or a copy of it if you want to customize without modifying the default).
3. Locate the **Browser - Conditional OTP** sub-flow.
4. Set the **Condition - User Configured** execution to **Required**.
5. Set the **OTP Form** execution to **Required**.

<Directive type="tip" title="Creating a Custom Flow">
It is recommended to duplicate the default Browser flow before modifying it. Click the three-dot menu next to the Browser flow and select **Duplicate**.
</Directive>

If you created a custom flow, bind it to the Realm:

1. Go to **Realm Settings** > **Authentication**.
2. Under **Browser flow**, select your custom flow.
3. Click **Save**.

## Step 3: Enforce MFA for Specific Users or Groups (Optional)

To require MFA only for a subset of users (for example, administrators), use a Conditional OTP policy:

1. Go to **Authentication** > **Flows** and open the custom Browser flow.
2. In the **Browser - Conditional OTP** sub-flow, set the condition to **Condition - Role**.
3. Specify the role that should be required to use OTP (for example, a `requires-mfa` Realm role).
4. Assign the `requires-mfa` role only to the users or groups that require MFA.

## Step 4: User OTP Enrollment

When a user logs in for the first time after MFA is enabled, Keycloak prompts them to enroll:

1. The user logs in with their username and password.
2. Keycloak displays a QR code for the authenticator app.
3. The user scans the QR code with their authenticator app.
4. The user enters the generated OTP code to confirm enrollment.
5. From the next login onward, the user is prompted to enter an OTP code.

## Verify MFA Is Working

1. Log out of the Keycloak Admin Console.
2. Log in as a test user in the target Realm.
3. After entering the password, confirm that the OTP prompt appears.
4. Enter the code from the authenticator app and confirm successful login.
